# 2.0

Released April 2025

**For PocketMine-MP 5.42.X**

LibDB 2.0 is a complete extraction of the database layer from EasyLibrary's `imperazim\components\database` namespace into a standalone, independently versioned library under `imperazim\db`. This release decouples database functionality from the monolith so that plugins can depend on LibDB without pulling in the entire EasyLibrary ecosystem. The new namespace is shorter, the architecture is cleaner, and every class has been redesigned for standalone use with proper error handling through `DatabaseException` instead of the old `PluginException` that coupled the database layer to EasyLibrary internals.

---

## Core

### `imperazim\db\LibDB`

#### Highlights

- Added `LibDB` as the PocketMine-MP `PluginBase` entry point, replacing the monolithic EasyLibrary plugin dependency. This class exists solely to give PocketMine a plugin to load, ensuring LibDB can be installed as an independent `.phar` without requiring EasyLibrary on the server.
- Removed all component-registration logic that previously lived in EasyLibrary's `PluginToolkit`. LibDB no longer registers itself through a component system -- it simply loads at `STARTUP` and makes its classes available via PSR-4 autoloading.

#### Usage

```php
// No direct usage required -- LibDB is loaded automatically by PocketMine.
// Simply place LibDB.phar in your plugins/ folder and depend on it in plugin.yml:
//
//   depend: [LibDB]
```

### `imperazim\db\Database`

#### Highlights

- Added `Database` as the common interface that both `Sqlite3` and `Mysql` drivers implement. In EasyLibrary v1.x, drivers shared no formal contract -- each had its own method signatures. The interface enforces a consistent API across drivers, making it possible to swap backends without changing application code.
- Added `createTableIfNotExists(array $tables): void` -- accepts an associative array of table name to column definitions, creating tables only when they do not already exist.
- Added `insert(string $table, array $data): void` -- inserts a single row with column-value pairs.
- Added `select(string $table, string $columns, array $filters = []): array` -- selects rows with optional WHERE filters, returning an array of associative rows.
- Added `update(string $table, string $column, mixed $value, array $filters = []): bool` -- updates a single column's value for matching rows, returns whether any rows were affected.
- Added `delete(string $table, array $filters): int` -- deletes matching rows, returns the count. Requires non-empty filters to prevent accidental full-table deletes.
- Added `exists(string $table, array $conditions): bool` -- checks if at least one matching row exists without fetching data.
- Added `query(string $sql, array $params = []): array` -- executes raw SQL with parameter binding for cases where the CRUD methods are not flexible enough.
- Added `close(): void` -- explicitly closes the connection, important for freeing resources in long-running PocketMine servers.

#### Usage

```php
use imperazim\db\Database;
use imperazim\db\DBManager;

// The Database interface is what you type-hint against:
function fetchPlayers(Database $db): array {
    return $db->select('players', 'name, level', [['active' => 1]]);
}

$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);
$players = fetchPlayers($db);
$db->close();
```

### `imperazim\db\DBManager`

#### Highlights

- Added `DBManager` as a static factory replacing EasyLibrary's `DatabaseManager`. The old class lived inside `imperazim\components\database` and threw `PluginException` on failure, tightly coupling it to the plugin framework. `DBManager` throws `DatabaseException` instead, keeping the error hierarchy within the `imperazim\db` namespace.
- Changed `connect(string $type, array $config): Database` to use PHP 8.1's `match` expression instead of a `switch` block, making unsupported driver types a compile-time-safe error path.
- Changed SQLite instantiation to automatically extract the directory and filename from a single `database` path in the config array, matching how MySQL config works. In v1.x, SQLite required separate `directory` and `fileName` keys.

#### Usage

```php
use imperazim\db\DBManager;
use imperazim\db\Database;
use imperazim\db\exception\DatabaseException;

// SQLite -- just pass the full path (without .db extension)
$sqlite = DBManager::connect('sqlite', [
    'database' => '/path/to/server/plugin_data/mydb'
]);

// MySQL
$mysql = DBManager::connect('mysql', [
    'host'     => 'localhost',
    'username' => 'root',
    'password' => 'secret',
    'database' => 'minecraft'
]);

// Error handling
try {
    $db = DBManager::connect('postgres', []);
} catch (DatabaseException $e) {
    // "Unsupported database type: postgres"
}
```

### `imperazim\db\Sqlite3`

#### Highlights

- Added `Sqlite3` as the standalone SQLite driver implementing `Database`. Replaces the embedded SQLite logic that previously existed as part of EasyLibrary's database component. Uses PDO with `ERRMODE_EXCEPTION` and `FETCH_ASSOC` defaults for consistent, safe operation.
- Added `upsert(string $table, array $data): void` -- performs `INSERT OR REPLACE` for atomic insert-or-update operations, a common need in player data storage where you want to create a record if it does not exist or update it if it does.
- Added `transaction(Closure $callback): void` -- wraps a callback in `BEGIN`/`COMMIT` with automatic `ROLLBACK` on exception. Essential for operations that modify multiple tables atomically (e.g., transferring currency between players).
- Added `getPdo(): PDO` -- exposes the underlying PDO instance for advanced use cases like custom pragmas or attaching databases.
- Changed all error handling from `new \crashdump($e)` (EasyLibrary v1.x pattern) to throwing `DatabaseException` with the original exception chained, providing proper stack traces.
- Changed identifier quoting to use backtick escaping (`quoteName()`) on all table and column names, preventing SQL injection through identifier manipulation.
- Changed all value binding to use PDO prepared statements exclusively -- no string interpolation of user values anywhere in the driver.

#### Usage

```php
use imperazim\db\Sqlite3;
use imperazim\db\exception\DatabaseException;

// Direct instantiation (DBManager is preferred, but direct works too)
$db = new Sqlite3('/path/to/data', 'players');

// Create tables
$db->createTableIfNotExists([
    'players' => [
        'id INTEGER PRIMARY KEY AUTOINCREMENT' => '',
        'name TEXT NOT NULL UNIQUE'             => '',
        'level INTEGER DEFAULT 1'              => '',
        'coins REAL DEFAULT 0.0'               => '',
    ]
]);

// CRUD
$db->insert('players', ['name' => 'Steve', 'level' => 5, 'coins' => 100.0]);

$players = $db->select('players', 'name, level', [['level' => 5]]);

$db->update('players', 'level', 10, [['name' => 'Steve']]);

$deleted = $db->delete('players', [['name' => 'Steve']]);

$exists = $db->exists('players', [['name' => 'Alex']]);

// Upsert (insert or replace)
$db->upsert('players', ['name' => 'Steve', 'level' => 15, 'coins' => 500.0]);

// Transactions
$db->transaction(function (Sqlite3 $db) {
    $db->update('players', 'coins', 50.0, [['name' => 'Steve']]);
    $db->update('players', 'coins', 150.0, [['name' => 'Alex']]);
});

// Raw query with parameter binding
$results = $db->query('SELECT * FROM `players` WHERE `level` > ?', [10]);

// Access underlying PDO
$pdo = $db->getPdo();
$pdo->exec('PRAGMA journal_mode=WAL');

$db->close();
```

### `imperazim\db\Mysql`

#### Highlights

- Added `Mysql` as the standalone MySQL driver implementing `Database`. Replaces the embedded MySQL logic from EasyLibrary. Uses `mysqli` with prepared statements and automatic type detection for parameter binding.
- Added static `connect(string $host, string $user, string $password, string $database): self` -- factory method that creates the `mysqli` connection and wraps it, throwing `DatabaseException` on failure instead of returning error codes.
- Added `upsert(string $table, array $data): void` -- performs `INSERT ... ON DUPLICATE KEY UPDATE` for MySQL-native upsert behavior. Unlike SQLite's `INSERT OR REPLACE` (which deletes then re-inserts), MySQL's approach preserves auto-increment IDs and triggers.
- Added `transaction(Closure $callback): void` -- wraps a callback in `begin_transaction()`/`commit()` with automatic `rollback()` on exception.
- Added `getMysqli(): mysqli` -- exposes the underlying connection for advanced operations like `multi_query()` or server status checks.
- Added automatic type string builder (`buildTypeString()`) that maps PHP types to mysqli bind types: `int` to `i`, `float` to `d`, everything else to `s`. This eliminates the manual type string construction that was error-prone in v1.x.
- Changed all error handling to throw `DatabaseException` with chained original exceptions.
- Changed identifier quoting to use the same `quoteName()` backtick-escaping approach as `Sqlite3` for consistency.

#### Usage

```php
use imperazim\db\Mysql;
use imperazim\db\exception\DatabaseException;

// Connect via static factory
$db = Mysql::connect('localhost', 'root', 'password', 'minecraft');

// Create tables
$db->createTableIfNotExists([
    'players' => [
        'id INT AUTO_INCREMENT PRIMARY KEY' => '',
        'name VARCHAR(64) NOT NULL UNIQUE'  => '',
        'level INT DEFAULT 1'               => '',
    ]
]);

// CRUD (same interface as Sqlite3)
$db->insert('players', ['name' => 'Steve', 'level' => 5]);

$players = $db->select('players', '*', [['level' => 5]]);

$updated = $db->update('players', 'level', 10, [['name' => 'Steve']]);

$deleted = $db->delete('players', [['name' => 'Steve']]);

// Upsert (ON DUPLICATE KEY UPDATE)
$db->upsert('players', ['name' => 'Steve', 'level' => 15]);

// Transactions
$db->transaction(function (Mysql $db) {
    $db->insert('players', ['name' => 'Alex', 'level' => 1]);
    $db->update('players', 'level', 99, [['name' => 'Alex']]);
});

// Raw queries
$high = $db->query('SELECT * FROM `players` WHERE `level` > ?', [50]);

// Access underlying mysqli
$mysqli = $db->getMysqli();

$db->close();
```

## Query Builder

### `imperazim\db\QueryBuilder`

#### Highlights

- Added `QueryBuilder` as a fluent, safe SQL construction layer. In EasyLibrary v1.x, complex queries required manual SQL string concatenation. `QueryBuilder` provides a chainable API that handles parameter binding, identifier quoting, and SQL injection prevention automatically.
- Added `table(Database $db, string $table): self` -- static factory that creates a builder bound to a specific table and connection.
- Added `select(string $columns = '*'): self` -- sets columns for SELECT queries.
- Added `where(string $column, string $operator, mixed $value): self` -- adds a WHERE condition with operator validation. Only allows safe operators: `=`, `!=`, `<>`, `>`, `<`, `>=`, `<=`, `LIKE`, `NOT LIKE`. Throws `DatabaseException` on invalid operators to prevent SQL injection through operator manipulation.
- Added `whereNull(string $column): self` -- adds a `WHERE column IS NULL` condition.
- Added `whereNotNull(string $column): self` -- adds a `WHERE column IS NOT NULL` condition.
- Added `orderBy(string $column, string $direction = 'ASC'): self` -- sets ORDER BY with direction validation.
- Added `limit(int $count): self` -- sets LIMIT with non-negative enforcement.
- Added `offset(int $count): self` -- sets OFFSET for pagination.
- Added `get(): array` -- executes the built SELECT query and returns result rows.
- Added `first(): ?array` -- convenience method that sets LIMIT 1 and returns the first row or null.
- Added `count(): int` -- executes `SELECT COUNT(*)` with the current WHERE conditions.
- Added `insert(array $data): void` -- inserts a row through the builder.
- Added `update(array $data): int` -- updates matching rows with multiple column-value pairs at once, unlike the `Database` interface which updates one column at a time.
- Added `deleteRows(): int` -- deletes matching rows. Throws `DatabaseException` if no WHERE conditions are set, preventing accidental full-table deletes.
- Added `exists(): bool` -- checks if any matching row exists.

#### Usage

```php
use imperazim\db\QueryBuilder;
use imperazim\db\DBManager;

$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);

// SELECT with conditions, ordering, and pagination
$players = QueryBuilder::table($db, 'players')
    ->select('name, level')
    ->where('level', '>', 10)
    ->where('active', '=', 1)
    ->orderBy('level', 'DESC')
    ->limit(20)
    ->offset(0)
    ->get();

// Get first matching record
$admin = QueryBuilder::table($db, 'players')
    ->select('*')
    ->where('name', '=', 'Admin')
    ->first();

// Count matching rows
$highLevel = QueryBuilder::table($db, 'players')
    ->where('level', '>=', 50)
    ->count();

// NULL checks
$unnamed = QueryBuilder::table($db, 'players')
    ->whereNull('nickname')
    ->get();

$named = QueryBuilder::table($db, 'players')
    ->whereNotNull('nickname')
    ->get();

// Check existence
$hasAdmin = QueryBuilder::table($db, 'players')
    ->where('name', '=', 'Admin')
    ->exists();

// Insert through builder
QueryBuilder::table($db, 'players')
    ->insert(['name' => 'Notch', 'level' => 100]);

// Update multiple columns at once
QueryBuilder::table($db, 'players')
    ->where('name', '=', 'Notch')
    ->update(['level' => 99, 'coins' => 1000]);

// Delete with safety (throws without WHERE)
QueryBuilder::table($db, 'players')
    ->where('level', '<', 1)
    ->deleteRows();
```

## Async

### `imperazim\db\async\AsyncQuery`

#### Highlights

- Added `AsyncQuery` as the public API for running non-blocking database queries. PocketMine-MP runs on a single thread, and synchronous database calls block the game loop. `AsyncQuery` dispatches queries to PocketMine's `AsyncPool` worker threads, returning results via callbacks on the main thread.
- Added `sqlite(string $dbPath, string $sql, array $params = [], ?Closure $onComplete = null, ?Closure $onError = null): void` -- runs a SQLite query asynchronously. The `$dbPath` is the full path to the `.db` file, and callbacks are invoked on the main thread after the query completes.
- Added `mysql(array $config, string $sql, array $params = [], ?Closure $onComplete = null, ?Closure $onError = null): void` -- runs a MySQL query asynchronously with the same callback pattern.
- Changed the async approach from EasyLibrary v1.x, which passed closures to worker threads (fragile and not serializable). The new design serializes only the SQL, parameters, and connection config -- all of which are plain data. Closures stay on the main thread via `storeLocal()`.

#### Usage

```php
use imperazim\db\async\AsyncQuery;

// Async SQLite query
AsyncQuery::sqlite(
    '/path/to/server/plugin_data/mydb.db',
    'SELECT * FROM players WHERE level > ?',
    [10],
    onComplete: function (array $rows): void {
        foreach ($rows as $row) {
            echo $row['name'] . ' is level ' . $row['level'] . PHP_EOL;
        }
    },
    onError: function (\Throwable $e): void {
        echo 'Query failed: ' . $e->getMessage() . PHP_EOL;
    }
);

// Async MySQL query
AsyncQuery::mysql(
    ['host' => 'localhost', 'username' => 'root', 'password' => '', 'database' => 'mc'],
    'UPDATE players SET last_seen = NOW() WHERE name = ?',
    ['Steve'],
    onComplete: function (array $rows): void {
        echo 'Player updated.' . PHP_EOL;
    }
);

// Fire-and-forget (no callbacks)
AsyncQuery::sqlite('/data/stats.db', 'INSERT INTO log (event) VALUES (?)', ['player_join']);
```

### `imperazim\db\async\AsyncDatabaseTask`

#### Highlights

- Added `AsyncDatabaseTask` as the internal `AsyncTask` implementation that powers `AsyncQuery`. This class is not intended for direct use -- prefer `AsyncQuery` -- but is public for advanced use cases where you need custom task scheduling.
- Added constructor accepting `string $driverType`, `array $config`, `string $sql`, `array $params`, and optional `?Closure $onComplete` and `?Closure $onError` callbacks. Config and params are serialized for safe transport to the worker thread; closures are stored via `storeLocal()` for main-thread-only access.
- Added `onRun(): void` -- executes on the worker thread. Creates a fresh database connection (PDO for SQLite, mysqli for MySQL), runs the prepared statement, and stores the result. Each query gets its own connection to avoid thread-safety issues.
- Added `onCompletion(): void` -- executes on the main thread. Retrieves the stored callbacks and invokes `onComplete` with the result rows on success, or `onError` with a `RuntimeException` on failure.
- Changed error handling to catch all `Throwable` in the worker thread and pass error messages back to the main thread as serialized data, preventing uncaught exceptions from crashing worker threads.

#### Usage

```php
use imperazim\db\async\AsyncDatabaseTask;
use pocketmine\Server;

// Advanced: manually submit an async task
$task = new AsyncDatabaseTask(
    'sqlite',
    ['database' => '/data/mydb.db'],
    'SELECT COUNT(*) AS total FROM players',
    [],
    onComplete: function (array $rows): void {
        $total = $rows[0]['total'] ?? 0;
        echo "Total players: $total" . PHP_EOL;
    },
    onError: function (\Throwable $e): void {
        echo 'Failed: ' . $e->getMessage() . PHP_EOL;
    }
);

Server::getInstance()->getAsyncPool()->submitTask($task);
```

## Cache

### `imperazim\db\cache\CacheLayer`

#### Highlights

- Added `CacheLayer` as an in-memory query cache with TTL expiration and automatic table-level invalidation. PocketMine plugins frequently read the same data every tick (e.g., player stats for HUD display). `CacheLayer` eliminates redundant database round-trips by caching results and automatically invalidating them when the underlying table is modified.
- Added constructor `__construct(Database $db, int $defaultTtl = 60)` -- wraps any `Database` instance with a configurable default TTL in seconds.
- Added `remember(string $key, ?int $ttl, Closure $loader): mixed` -- the primary caching method. Returns cached data if the key exists and has not expired; otherwise calls the loader, caches the result, and returns it.
- Added `put(string $key, mixed $data, ?int $ttl = null): void` -- manually stores a value in the cache.
- Added `has(string $key): bool` -- checks if a non-expired entry exists.
- Added `get(string $key, mixed $default = null): mixed` -- retrieves a cached value with a default fallback.
- Added `forget(string $key): void` -- removes a specific cache entry.
- Added `tag(string $table, string $key): void` -- associates a cache key with a table name for automatic invalidation when that table is modified.
- Added `select(string $table, string $columns = '*', array $filters = [], ?int $ttl = null): array` -- cached version of `Database::select()` that automatically tags the cache key to the table.
- Added `insert(string $table, array $data): void` -- delegates to the database and automatically invalidates all cache entries tagged to that table.
- Added `update(string $table, string $column, mixed $value, array $filters = []): bool` -- delegates to the database and invalidates the table's cache.
- Added `delete(string $table, array $filters): int` -- delegates to the database and invalidates the table's cache.
- Added `invalidateTable(string $table): void` -- manually invalidates all cache entries associated with a table.
- Added `flush(): void` -- clears all cached data and table tags.
- Added `stats(): array` -- returns cache statistics (`entries` count and `tables` count) after cleaning up expired entries.
- Added `getDatabase(): Database` -- returns the underlying database instance for operations that should bypass the cache.

#### Usage

```php
use imperazim\db\cache\CacheLayer;
use imperazim\db\DBManager;

$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);
$cache = new CacheLayer($db, defaultTtl: 30);

// Cached select -- subsequent calls within 30s return cached data
$players = $cache->select('players', '*', [['active' => 1]]);

// Custom TTL per query
$leaderboard = $cache->select('players', 'name, level', [], ttl: 120);

// Generic remember pattern for arbitrary data
$totalCoins = $cache->remember('stats:total_coins', 60, function () use ($db) {
    $result = $db->query('SELECT SUM(coins) AS total FROM players');
    return $result[0]['total'] ?? 0;
});

// Write-through operations (automatically invalidate related cache)
$cache->insert('players', ['name' => 'Steve', 'level' => 1]);
$cache->update('players', 'level', 50, [['name' => 'Steve']]);
$cache->delete('players', [['name' => 'Steve']]);

// Manual cache control
$cache->forget('stats:total_coins');
$cache->invalidateTable('players');
$cache->flush();

// Inspect cache state
$stats = $cache->stats(); // ['entries' => 5, 'tables' => 2]

// Bypass cache for specific operations
$raw = $cache->getDatabase()->query('PRAGMA table_info(players)');
```

## Migration

### `imperazim\db\migration\MigrationStep`

#### Highlights

- Added `MigrationStep` as the abstract base class for individual migration definitions. Each migration step defines an `up()` method to apply schema changes and a `down()` method to revert them. This two-way design enables both forward migration and rollback, which was not possible in EasyLibrary v1.x where schema changes were one-way `createTableIfNotExists()` calls scattered throughout plugin startup code.

#### Usage

```php
use imperazim\db\migration\MigrationStep;
use imperazim\db\Database;

class CreatePlayersTable extends MigrationStep {
    public function up(Database $db): void {
        $db->createTableIfNotExists([
            'players' => [
                'id INTEGER PRIMARY KEY AUTOINCREMENT' => '',
                'name TEXT NOT NULL UNIQUE'             => '',
                'level INTEGER DEFAULT 1'              => '',
                'coins REAL DEFAULT 0.0'               => '',
                'created_at TEXT NOT NULL'              => '',
            ]
        ]);
    }

    public function down(Database $db): void {
        $db->query('DROP TABLE IF EXISTS `players`');
    }
}

class AddPlayerNickname extends MigrationStep {
    public function up(Database $db): void {
        $db->query('ALTER TABLE `players` ADD COLUMN `nickname` TEXT DEFAULT NULL');
    }

    public function down(Database $db): void {
        // SQLite does not support DROP COLUMN before 3.35.0;
        // recreate table without the column if needed.
        $db->query('ALTER TABLE `players` DROP COLUMN `nickname`');
    }
}
```

### `imperazim\db\migration\Migration`

#### Highlights

- Added `Migration` as the static migration runner and registry. Manages a `_migrations` tracking table that records which migrations have been applied and when. This replaces the ad-hoc "check if table exists, then maybe alter it" pattern from EasyLibrary v1.x with a proper versioned schema management system.
- Added `register(string $name, MigrationStep $step): void` -- registers a named migration step. Names should be sortable (e.g., `001_create_players`, `002_add_nickname`) to ensure consistent ordering. Duplicate registrations update the step without duplicating the order entry.
- Added `runAll(Database $db): string[]` -- runs all pending migrations in registration order, returning the names of migrations that were applied. Skips already-applied migrations automatically.
- Added `run(Database $db, string $name): bool` -- runs a single specific migration by name. Returns `false` if already applied, `true` if newly applied. Throws `DatabaseException` if the migration name is not registered.
- Added `rollback(Database $db, int $count = 1): string[]` -- rolls back the last N applied migrations in reverse order. Calls each step's `down()` method and removes the tracking record.
- Added `getApplied(Database $db): string[]` -- returns a list of applied migration names ordered by application time.
- Added `getPending(Database $db): string[]` -- returns a list of registered but not-yet-applied migration names.
- Added `resetAll(Database $db): string[]` -- rolls back all applied migrations in reverse order and returns their names. Useful for testing or full schema rebuilds.
- Added `clearRegistry(): void` -- clears all registered migrations from memory without touching the database. Useful for test isolation.

#### Usage

```php
use imperazim\db\migration\Migration;
use imperazim\db\migration\MigrationStep;
use imperazim\db\Database;
use imperazim\db\DBManager;

// Register migrations (typically in plugin onEnable)
Migration::register('001_create_players', new class extends MigrationStep {
    public function up(Database $db): void {
        $db->createTableIfNotExists([
            'players' => [
                'id INTEGER PRIMARY KEY AUTOINCREMENT' => '',
                'name TEXT NOT NULL UNIQUE'             => '',
                'level INTEGER DEFAULT 1'              => '',
            ]
        ]);
    }
    public function down(Database $db): void {
        $db->query('DROP TABLE IF EXISTS `players`');
    }
});

Migration::register('002_add_coins', new class extends MigrationStep {
    public function up(Database $db): void {
        $db->query('ALTER TABLE `players` ADD COLUMN `coins` REAL DEFAULT 0.0');
    }
    public function down(Database $db): void {
        $db->query('ALTER TABLE `players` DROP COLUMN `coins`');
    }
});

$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);

// Run all pending migrations
$applied = Migration::runAll($db);
// $applied = ['001_create_players', '002_add_coins']

// Check status
$pending = Migration::getPending($db);
$done = Migration::getApplied($db);

// Run a single migration
Migration::run($db, '002_add_coins');

// Rollback the last migration
$rolledBack = Migration::rollback($db, 1);
// $rolledBack = ['002_add_coins']

// Rollback everything
Migration::resetAll($db);

// Clear in-memory registry
Migration::clearRegistry();
```

## Model (Active Record ORM)

### `imperazim\db\model\Model`

#### Highlights

- Added `Model` as an abstract Active Record base class. EasyLibrary v1.x had no ORM -- plugins worked directly with arrays from `select()`. `Model` provides object-oriented record access with dirty tracking, automatic save/insert detection, and a fluent query interface. This dramatically reduces boilerplate in plugins that manage player data, economy records, or other persistent entities.
- Added `setDatabase(Database $db): void` -- static method to set the shared database connection for all models. Must be called once during plugin startup.
- Added `db(): Database` -- returns the current connection, throwing `DatabaseException` if none has been set.
- Added `getTable(): string` -- returns the model's table name (defined by subclasses via `protected static string $table`).
- Added `hydrate(array $attributes, bool $exists = true): static` -- creates a model instance from raw row data with its `exists` flag set. Used internally by query methods and available for manual hydration.
- Added `find(mixed $id): ?static` -- finds a record by primary key, returning null if not found.
- Added `findOrFail(mixed $id): static` -- finds a record by primary key, throwing `DatabaseException` if not found. Useful for guaranteed lookups.
- Added `all(): static[]` -- returns all records in the table as model instances.
- Added `where(string $column, string $operator, mixed $value): ModelQuery` -- starts a scoped query chain that returns model instances instead of raw arrays.
- Added `create(array $data): static` -- inserts a new record and returns a hydrated model instance.
- Added `query(): QueryBuilder` -- returns a raw `QueryBuilder` for the model's table, for cases where `ModelQuery` is too limiting.
- Added `save(): void` -- saves the model. If it already exists in the database, only dirty (changed) attributes are updated. If it is new, all attributes are inserted. The dirty tracking compares current attributes against the original snapshot taken at hydration time.
- Added `destroy(): bool` -- deletes the record from the database and marks the model as non-existent. Returns false if the model was not persisted.
- Added `getKey(): mixed` -- returns the primary key value.
- Added `getDirty(): array` -- returns an array of attributes that have changed since the model was loaded.
- Added `exists(): bool` -- returns whether the record exists in the database.
- Added `toArray(): array` -- returns all attributes as a plain associative array.
- Added `fill(array $data): static` -- mass-assigns attributes from an array.
- Added magic `__get`, `__set`, `__isset`, `__unset` for natural property-style attribute access (e.g., `$user->name` instead of `$user->getAttribute('name')`).

#### Usage

```php
use imperazim\db\model\Model;
use imperazim\db\DBManager;

// Define a model
class Player extends Model {
    protected static string $table = 'players';
    protected static string $primaryKey = 'id';
}

// Set up the database connection (once, during plugin startup)
$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);
Model::setDatabase($db);

// Create a new player
$player = Player::create([
    'name'  => 'Steve',
    'level' => 1,
    'coins' => 100.0,
]);

// Find by primary key
$player = Player::find(1);
$player = Player::findOrFail(1); // throws if not found

// Access attributes as properties
echo $player->name;   // 'Steve'
echo $player->level;  // 1

// Modify and save (only dirty fields are updated)
$player->level = 50;
$player->coins = 999.0;
$player->save();

// Check dirty state
$player->name = 'Herobrine';
$dirty = $player->getDirty(); // ['name' => 'Herobrine']

// Get all players
$everyone = Player::all();

// Mass fill attributes
$player->fill(['level' => 99, 'coins' => 0]);
$player->save();

// Convert to array
$data = $player->toArray();

// Delete
$player->destroy();

// Check existence
if ($player->exists()) {
    // still in database
}
```

### `imperazim\db\model\ModelQuery`

#### Highlights

- Added `ModelQuery` as a query builder scoped to a specific `Model` class. When you call `Model::where()`, you get a `ModelQuery` that knows which model class to hydrate results into. This bridges the gap between the raw `QueryBuilder` (which returns arrays) and the ORM layer (which returns model instances).
- Added `where(string $column, string $operator, mixed $value): self` -- adds a WHERE condition, chainable for multiple conditions.
- Added `orderBy(string $column, string $direction = 'ASC'): self` -- sets ORDER BY.
- Added `limit(int $count): self` -- sets LIMIT.
- Added `get(): Model[]` -- executes the query and returns an array of hydrated model instances.
- Added `first(): ?Model` -- returns the first matching model or null.
- Added `count(): int` -- returns the count of matching rows.
- Added `exists(): bool` -- checks if any matching row exists.
- Added `delete(): int` -- deletes matching rows directly without loading models.
- Added `update(array $data): int` -- updates matching rows directly without loading models.

#### Usage

```php
use imperazim\db\model\Model;
use imperazim\db\DBManager;

class Player extends Model {
    protected static string $table = 'players';
    protected static string $primaryKey = 'id';
}

Model::setDatabase(DBManager::connect('sqlite', ['database' => '/data/mydb']));

// Chain WHERE conditions
$pros = Player::where('level', '>=', 50)
    ->where('coins', '>', 1000)
    ->orderBy('level', 'DESC')
    ->limit(10)
    ->get();

// Get first match
$admin = Player::where('name', '=', 'Admin')->first();

// Count matching records
$newbies = Player::where('level', '<', 5)->count();

// Check existence
$hasAdmin = Player::where('name', '=', 'Admin')->exists();

// Bulk update without loading models
Player::where('level', '<', 1)
    ->update(['level' => 1]);

// Bulk delete without loading models
Player::where('coins', '<', 0)
    ->delete();
```

## Seed

### `imperazim\db\seed\Seeder`

#### Highlights

- Added `Seeder` for populating tables with initial or default data. In EasyLibrary v1.x, plugins manually checked if rows existed before inserting defaults, leading to duplicated and fragile startup code. `Seeder` provides multiple strategies for safe, repeatable data insertion.
- Added `run(Database $db, string $table, array $rows): int` -- inserts all provided rows unconditionally. Returns the count of inserted rows. Best used when the table is known to be empty or when duplicates are acceptable.
- Added `runOnce(Database $db, string $seedName, string $table, array $rows): int` -- inserts rows only once per server session, tracked by a unique seed name. Returns 0 on subsequent calls. Useful for data that should be seeded once per server restart but does not need persistent tracking.
- Added `runIfEmpty(Database $db, string $table, array $rows): int` -- inserts rows only if the table currently has zero rows. Returns 0 if the table already has data. The safest option for default data that should not be duplicated across server restarts.
- Added `refresh(Database $db, string $table, array $rows): int` -- truncates the table (DELETE all rows) and re-seeds with the provided data. Useful for reset operations or test fixtures.
- Added `isApplied(string $seedName): bool` -- checks if a named seed has been applied this session.
- Added `reset(): void` -- clears the in-memory applied seeds tracker, allowing `runOnce` seeds to be re-applied.

#### Usage

```php
use imperazim\db\seed\Seeder;
use imperazim\db\DBManager;

$db = DBManager::connect('sqlite', ['database' => '/data/mydb']);

// Seed default items (only if table is empty)
Seeder::runIfEmpty($db, 'items', [
    ['name' => 'Wooden Sword', 'damage' => 5,  'price' => 10],
    ['name' => 'Stone Sword',  'damage' => 10, 'price' => 25],
    ['name' => 'Iron Sword',   'damage' => 20, 'price' => 50],
]);

// Seed once per session (e.g., daily rewards reset)
Seeder::runOnce($db, 'daily_rewards_2025', 'rewards', [
    ['player' => '*', 'item' => 'Diamond', 'quantity' => 1],
]);

// Check if seed was already applied
if (Seeder::isApplied('daily_rewards_2025')) {
    echo 'Daily rewards already seeded this session.' . PHP_EOL;
}

// Force re-seed (truncate + insert)
Seeder::refresh($db, 'shop_defaults', [
    ['item' => 'Health Potion', 'price' => 50],
    ['item' => 'Speed Potion',  'price' => 75],
]);

// Unconditional insert (when you know the table is empty)
Seeder::run($db, 'logs', [
    ['event' => 'server_start', 'timestamp' => date('Y-m-d H:i:s')],
]);

// Reset session tracker (allows runOnce to fire again)
Seeder::reset();
```

## Exception

### `imperazim\db\exception\DatabaseException`

#### Highlights

- Added `DatabaseException` as the single exception type for all LibDB errors. Extends `RuntimeException` so it is unchecked and does not require explicit catches at every call site, but can be caught when needed.
- Changed from EasyLibrary v1.x's `PluginException` (which lived in `imperazim\components` and coupled database errors to the plugin framework) to a standalone exception within the `imperazim\db\exception` namespace. This means plugins that catch database errors no longer need to import anything from EasyLibrary.
- Changed from the v1.x pattern of `new \crashdump($e)` (which wrote crash dumps for every database error) to standard exception propagation. Database errors are now thrown normally and can be caught, logged, or re-thrown as the plugin developer sees fit.

#### Usage

```php
use imperazim\db\exception\DatabaseException;
use imperazim\db\DBManager;

try {
    $db = DBManager::connect('sqlite', ['database' => '/invalid/path/db']);
} catch (DatabaseException $e) {
    echo 'Database error: ' . $e->getMessage() . PHP_EOL;

    // Access the original driver exception if needed
    $previous = $e->getPrevious();
    if ($previous !== null) {
        echo 'Caused by: ' . $previous->getMessage() . PHP_EOL;
    }
}

// Works with all LibDB operations
try {
    $db->delete('players', []); // empty filters
} catch (DatabaseException $e) {
    // "Cannot delete without conditions. Use query() for raw DELETE."
}
```

## Migration Guide from EasyLibrary v1.x

| v1.x (EasyLibrary) | v2.0 (LibDB) |
|---|---|
| `imperazim\components\database\DatabaseManager` | `imperazim\db\DBManager` |
| `imperazim\components\database\Sqlite3` | `imperazim\db\Sqlite3` |
| `imperazim\components\database\Mysql` | `imperazim\db\Mysql` |
| `PluginException` on connect failure | `DatabaseException` on connect failure |
| `new \crashdump($e)` error handling | `throw new DatabaseException(...)` with chained cause |
| No interface for drivers | `imperazim\db\Database` interface |
| No query builder | `imperazim\db\QueryBuilder` |
| No async queries | `imperazim\db\async\AsyncQuery` |
| No cache layer | `imperazim\db\cache\CacheLayer` |
| No migration system | `imperazim\db\migration\Migration` |
| No ORM / Active Record | `imperazim\db\model\Model` |
| No seeder | `imperazim\db\seed\Seeder` |
| `depend: [EasyLibrary]` | `depend: [LibDB]` |
